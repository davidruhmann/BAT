Template job. Do not build! Do not edit!

Visual C++ and C# Builder Job. Builds the Release and Debug configurations for a project along with optional Code Analysis. An optional Klocwork analysis can be run on the Debug configuration and Doxygen generated for the project.

Usage Instructions:
  • Build > Use builders from another project > VCBuild
  • Post-build Actions > Use publishers from another project > VCBuild
  • Set any Environment Variables if non-defaults are desired
  • Defaults are overwritten based upon Node Labels specified.
Supported Versions:
  • VC 9.0 - Visual Studio 2008 (for MillenniumMobile Only)
  • VC 10.0 - Visual Studio 2010
  • VC 12.0 - Visual Studio 2013
Supported Labels:
  • VS2008 - Visual Studio 2008 compatible node. This will default VCVER to 90.
  • VS2010 - Visual Studio 2010 compatible node. This will default VCVER to 100.
  • VS2013 - Visual Studio 2013 compatible node. This will default VCVER to 130.
  • KLOCWORK - Klocwork v10 compatible node. This will default KLOCWORK to YES.
Optional Variables: (Defaults are bold)
  • VCARC - [x86|x64|...] - The target processor architecture.
  • VCVER - [90|100|120] - The compiler version to use.
  • VCLOG - [q[uiet]|m[inimal]|n[ormal]|d[etailed]|diag[nostic]] - The verbosity of the logging.
  • VCCON - [Release;Debug] - Semi-colon delimited list of configurations to build.
  • VCCAN - [NO|YES] - Run Microsoft Code Analysis on the project.
  • VCCAR - [*RecommendedRules.ruleset] Code Analysis Rule Set to use.
  • KLOCWORK - [NO|YES] - Run Klocwork analysis on the project?
  • DOXYGEN - [NO|YES] - Run Doxygen on the project?
Build Variables:
  • INCLUDE - Search list for header (.h) files.
  • LIB - Search list for static library (.lib) and type library (.tlb) files.
  • LIBPATH - (a.k.a. Reference Path) Search list for dynamic library (.dll) files for import.
  • PATH - (a.k.a. Executable Path) Search list for dynamic library (.dll) files for reference.
Klocwork Details:
  • New issues will make a trunk job unstable and will fail branch jobs.
See the CopyEndStates and CopyReports jobs for any other details.
